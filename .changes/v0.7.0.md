## v0.7.0 - 2026-06-27
### Added
* Added self-contained Queso release archives for Linux (glibc and musl), macOS, and Windows x86_64 targets while keeping the existing escript artifacts. Linux static targets are excluded because Queso static binaries do not export the NIF symbols (crypto/SSL) required by this CLI.
* SBOM components for git-sourced dependencies are now enriched from the dependency's locally checked-out `gleam.toml` (`build/packages/<name>/gleam.toml`), adding description, licences, and links that previously only Hex packages received. The repository link is taken from the manifest git URL so it matches the component purl even when a dependency is sourced from a fork. When the local source is unavailable the component still carries a repository link and a warning records the omitted fields.
* Added `just ort-*` recipes (ort-analyze, ort-report, ort-sbom, ort-compare, ort-clean) that generate a reference CycloneDX SBOM with the OSS Review Toolkit to cross-check ours, plus docs/sbom-comparison-ort.md documenting the comparison.
### Fixed
* SBOM generation no longer silently drops package enrichment (description, publisher, licences, and links) when a Hex metadata fetch fails. The licence cache now falls back to a stale cached entry when a refetch fails, and both the stale fallback and any package left without metadata are surfaced as clear warnings that include the underlying failure reason (connection, TLS, or timeout).
